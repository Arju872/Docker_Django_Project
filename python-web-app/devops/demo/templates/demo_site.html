This page is a demo README-style visual for a project that packages a Django app into containers, shows how to run locally and describes DevOps features used in the demo.

Run locally with Docker

Build and run the Django app using a Dockerfile and docker-compose for local development and testing.

Persistent data & volumes

Use Docker volumes for database persistence (Postgres) and static/media storage for safe local iterations.

Environment parity

Containerization ensures parity between development, staging, and production minimizing "works on my machine" issues.

How to run (quick)

# build
docker build -t my-django-app:demo .
# or using docker-compose
docker-compose up --build
# visit http://localhost:8000

This demo assumes your Django settings are configured to read secrets from environment variables (DATABASE_URL, SECRET_KEY) and collectstatic is used for static files in production builds.

DevOps Features — At a glance

This demo project highlights common DevOps practices used when containerizing Django apps.

  • CI/CD pipeline: Example GitHub Actions or GitLab CI to run tests, build images, and push artifacts.
  • Image building: Multi-stage Dockerfile to produce small production images.
  • Secrets management: Environment variables, .env for local dev, and secret stores (Vault/KMS) for production.
  • Healthchecks: Container HEALTHCHECK and readiness endpoints for orchestrators.
  • Orchestration: Kubernetes / ECS manifests or Helm charts for deployment (example manifests included).
  • Observability: Basic logging to stdout, structured logs and optional metrics export (Prometheus).
  • Rollback: Tagged image deployments and simple rollback strategy for quick recovery.
This project is a demo for:
Containerization of a Django application — builds a Docker image from the Django codebase, runs migrations automatically on startup (optional), and serves the site behind a lightweight web server (Gunicorn). Designed for local development and to be extended for CI/CD and cloud deployment.

Example Dockerfile snippet

FROM python:3.11-slim\nWORKDIR /app\nCOPY requirements.txt ./\nRUN pip install -r requirements.txt\nCOPY . .\nCMD ["gunicorn","project.wsgi:application","--bind","0.0.0.0:8000"]
